ES6 syntax review

Since we started learning React we've been getting acquainted with ES6 syntax here and there. Let's dedicate sometime to an official overview.

For a more complete guide visit Ben Ilegbodu's Learning ES6 repository.

  1. const + let

    These replace the usage of var. A const cannot be reassigned a value whereas a let can. 

      // ES5
  var name = 'Maria';

      // ES6
  const name = 'Maria';

      // ES5
  var mood = 'happy';
  mood = 'cranky';

      // ES6
  let mood = 'happy';
  mood = 'cranky';

  2. Arrow functions

      // ES5
  function name (firstName, lastName) {

  }

      // ES6
  const name = (firstName, lastName) => {

  }

  3. Arrow functions with implicit return

    The main syntax difference for ES6 is that instead of {} we use (). We also have to make sure we don't include any semicolons inside of our implicit return.

      // ES5
  function (firstName, lastName) {
    return {foo: 'bar'};
  }

      // ES6
  (firstName, lastName) => ({foo: 'bar'});

  4. Destructuring

    Defines new variables (let or const) that exist as properties in an object.

      // ES5
  function getObj () {
    return {
      firstName: 'Maria',
      lastName: 'Tzeka'
    };
  };

  var firstName = getObj().firstName;
  var lastName = getObj().lastName;

      // ES6
  const getObj = () => ({
    fistName: 'Maria',
    lastName: 'Tzeka'
  });

  const {firstName, lastName} = getObj();

  5. Shorthand property assignment

    Assuming you have a variable defined with the same name as the property, you can use this nifty shorthand syntax.

      // ES5
  var firstName = 'Maria';
  var lastName = 'Tzeka';

  var obj = {
    firstName: firstname,
    lastName: lastName
  };

      // ES6
  const firstName = 'Maria';
  const lastName = 'Tzeka';

  const obj = {firstName, lastName};

  6. Template strings

    Instead of quotes we use a tick ` with js values wrapped around ${}.

      // ES5
  function sayMyName(firstName) {
    return 'Hello, ' + firstName + '.';
  }

      // ES6
  const sayMyName = firstName => (
    `Hello, ${firstName}.`
  );

localStorage v.s. sessionStorage

Two common ways of storing data on your browser are with localStorage and sessionStorage. localStorage persists even when you close the window and revisit, and sessionStorage persists only on refresh. As soon as you close the window the data is wiped out. localStorage has no expiration time, unless the user clears the browser cache on their own.

setItem

localStorage.setItem('foo', 'bar');
sessionStorage.setItem('foo', 'bar');

The method which defines an item from storage; first argument is the property name and second arguement is the value.

getItem

The method which grabs an item from storage.

localStorage.getItem('foo');
sessionStorage.getItem('foo');
// 'bar'

removeItem

Removes item when providing an argument of the property name.

localStorage.removeItem('foo');
sessionStorage.removeItem('foo');
