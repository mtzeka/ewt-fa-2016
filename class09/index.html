Stateful components

Stateful components are components with state. What is state? It is like props but private and meant to be maintained within the component. It is also the only way we can cause a component to re-render within itself.

A few things we have to do to create a stateful component:

  • Instead of a function we need to define an ES6 `class` that extends `React.Component`
  • Add a single method of `render`.
  • Instead of a `props` argument we access the props via `this.props` (so you basically just type `this.props` instead of `props`).

Updating state

Component state is an object just like props is. In order to set a value for state we need to pass an object. (review: objects are data types in javascript that start with are wrapped around `{}`. Read more about them here).

The method that allows us to update state is the following. However there's not much value in updating state in the render method, otherwise we would end up in an endless loop (remember, updating state forces a re-render).

this.setState({
  name: 'Maria'
});

Handling events

A perfect use case of when to update state is when you want the UI to update after a browser event. The most common being a click event.

The nice thing about React is you just pass an `onClick` prop to define the click event...!

To update state when clicking on an event, we just use `setState`. Notice we are also setting an initial state value at the very top of our component. This is the value that the component first renders with.

Other supported events

  • Clipboard: onCopy, onCut, onPaste
  • Keyboard: onKeyDown, onKeyPress, onKeyUp
  • Focus: onFocus, onBlur
  • Form: onChange, onInput, onSubmit
  • Click: onClick, onContextMenu, onDoubleClick, onDrag, onDragEnd, onDragEnter, onDragExit, onDragLeave, onDragOver, onDragStart, onDrop, onMouseDown, onMouseEnter, onMouseLeave, onMouseMove, onMouseOut, onMouseOver, onMouseUp
  • UI: onScroll
  • Image: onLoad, onError
  • Animation: onAnimationStart, onAnimationEnd, onAnimationIteration
