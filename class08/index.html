Intro to React

10.26.2016

What is React?

React is a JavaScript framework developed by Facebook that has become extremely popular and favorable in the past couple of years. It has unique features and syntax that has simplified the way we build large applications.

"React makes it painless to create interactive UIs. Design simple views for each state in your application, and React will efficiently update and render just the right components when your data changes." - React

What is JSX?

JSX is a syntax extension of JavaScript used in React. It looks like HTML but it is not. Under the hood it is creating React DOM elements in JavaScript. It is not natively supported by browsers and will need to use Babel to transpile it for us.

var element = <h1>Hello, world!</h1>;

// transpiled
var element = React.createElement("h1", null, "Hello World!");

Read more about it in the React documentation

Note: JSX isn't HTML so there will be some differences. For example you need to have a wrapping element in place at all times. You also need to use the "className" attribute instead of "class" (yes, casing matters).

// invalid
const element = (
  <h1 class="title">Hello, world!</h1>
  <h2>How are you?</h2>
);

// valid
const element = (
  <header>
    <h1 className="title">Hello, world!</h1>
    <h2>How are you?</h2>
  </header>
);

What is ES6?

ES is shorthand for EcmaScript, which is the trademarked language specification based on JavaScript standardized by Ecma International. At the moment browsers support ES5 which is the JavaScript we have all learned in class. ES6 is not fully implemented in browsers but developers have adapted it and write it in their local environments which then transpiles to ES5. ES6 is known to look a lot cleaner and easier to read -- in the past some developer were using CoffeeScript for cleaner syntax; which is not as widely used since ES6 has been introduced.

What is Babel?

Since ES6 is not fully supported in browsers, but developers like to write in it, just like they like to write sass instead of css. Legibility = happy programmers. Therefore we have to use a transpiler to convert ES6 to ES5. One of the most well known in the industry is Babel.

Babel isn't exactly a tool that works on its own, like sass. It's more of a plugin that gets added to your existing build environment. Most popular ways of using babel are via Gulp and webpack.

For real-time transpiling we can include babel in our HTML file with babel-standalone. Not exactly recomended for large applications since it is a lot slower than bundling.

<script src="https://unpkg.com/babel-standalone@6.15.0/babel.min.js"></script>

Setting up React in the browser

In a perfect world we'd have a development environment set up with all of our js minified and bundled into a single file. For class purposes we will import our libraries one by one.

<script src="https://unpkg.com/react@latest/dist/react.js"></script>
<script src="https://unpkg.com/react-dom@latest/dist/react-dom.js"></script>
<script src="https://unpkg.com/babel-standalone@6.15.0/babel.min.js"></script>

In the body at the very end we'll need to include the JavaScript (NOTE: we have to set the type to "text/babel" since we want it to transpile).

<script type="text/babel">
  // JS goes here
</script>

Functional components

Documentation

These are stateless components that are defined as functions.

function Welcome() {
  return (
    <h1>Hello World</h1>
  );
}

To render components to the DOM we use the "render" method off of "ReactDOM".

ReactDOM.render(<Welcome/>, document.querySelector('.wrapper'));

Props

In stateless components props are passed as arguments and defined as attributes when the component is used.

function Welcome(props) {
  return (
    <h1>Hello {props.name}</h1>
  );
}

function App() {
  return (
    <Welcome name="Maria"/>
  );
}

ReactDOM.render(<App/>, document.querySelector('.wrapper'));
