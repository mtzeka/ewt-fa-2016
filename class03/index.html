    • What are responsive images?

    Responsive images can be thought of as "smart" images. Unlike normal images they can be configured to display different images depending on the resolution of the page.

    If you're on a mobile device, would you want to download an image meant for a desktop device? Probably not. The image will be much larger and take longer to download.

    You can read more about them on MDN.

    How do we implement them?

    Responsive images just require us to add the "srcset" attribute to our existing image tag. We specify the pathname to the image, and also the size it displays in (notice we don't use "px" but we use "w").

    We still need to provide it with the original "src" image which is treated as the default fallback image.

    <img
      alt="A pretty picture"
      src="images/original.jpg"
      srcset="images/small.jpg 600w,
              images/medium.jpg 800w,
              images/large.jpg 1200w,
              images/extralarge.jpg 1600w"
    />

    Using Polyfills

    Before using responsive images in production be aware that they are not supported in IE. You will neex to usePicturefill in your project.

    After adding the file to your js directory the following code must be inserted in the <head> tag.

    <script>
      document.createElement("picture");
    </script>
    <script src="js/picturefill.min.js" async></script>
