Responsive Images

There are two different usages of responsive images:

  1. Resolution switching
  2. Art direction

NOTE: Remember to have your cache disabled for dev purposes!

1. Resolution Switching example

<img src="elva-fairy-800w.jpg" alt="Elva dressed as a fairy">

// Using different sizes (example)
<img
  srcset="elva-fairy-320w.jpg 320w,
    elva-fairy-480w.jpg 480w,
    elva-fairy-800w.jpg 800w"
  sizes="(max-width: 320px) 280px,
    (max-width: 480px) 440px,
    800px"
  src="elva-fairy-800w.jpg" alt="Elva dressed as a fairy"
>

// Using the same sizes but different resolutions (example)
<img
  srcset="elva-fairy-320w.jpg,
    elva-fairy-480w.jpg 1.5x,
    elva-fairy-640w.jpg 2x"
  src="elva-fairy-640w.jpg" alt="Elva dressed as a fairy"
>

2. Art direction example

<img src="elva-800w.jpg" alt="Chris standing up holding his daughter Elva">

// example
<picture>
  <source media="(max-width: 799px)" srcset="elva-480w-close-portrait.jpg">
  <source media="(min-width: 800px)" srcset="elva-800w.jpg">
  <img src="elva-800w.jpg" alt="Chris standing up holding his daughter Elva">
</picture>

    HW

    Download the homework zip from moodle and convert the css to scss.

    Guidelines:

    1. Define 6 variables (minimum). Create a variable for:
      • Colors -- eliminate repeated use of colors
      • Width/breakpoint values for table as well as desktop
      • Animations
      • Anything else that you notice is repeated
    2. Create two mixins for any css rules that are repeated from vendor prefixes. One of your mixins will need to take a variable argument.
    3. Nest rules Wherever you notice a child or a psuedo selector. Some will use the `&` for back-references.
